# [A1] Product Scope & Requirements Freeze
Define the final MVP feature set, supported iOS versions, and non-functional requirements (privacy, accessibility, localization scope).

**Outcomes**
- Locked MVP feature set.
- Documented platform support and non-functional requirements.
- Branding asset checklist finalized.

# [A2] Data Model & Persistence
Finalize the SwiftData model, migrations, and initial data handling.

**Outcomes**
- Complete `Flashcard` schema.
- Migration strategy documented.
- Persistence tests in place.

# [A3] Core UX & Navigation
Stabilize screens, navigation flows, and user journeys (list, create, edit, study).

**Outcomes**
- Core navigation map approved.
- Create/Edit and Study flows implemented.
- Empty states and onboarding prompts in place.

# [A4] UI Polish & Accessibility
Refine visuals, typography, and accessibility support.

**Outcomes**
- Design system applied.
- Accessibility labels, dynamic type, and VoiceOver support confirmed.
- Contrast and tap-target sizes validated.

# [A5] App Functionality Enhancements
Add optional but likely-needed features: import/export, search, tags, reminders.

**Outcomes**
- Search/filtering and tagging available.
- Import/export workflow implemented.
- Reminders (if desired) integrated.

# [A6] Privacy, Security & Compliance
Prepare privacy disclosures and data handling documentation.

**Outcomes**
- Data inventory completed.
- Privacy policy drafted and hosted.
- App Store privacy questionnaire responses prepared.

# [A7] Testing, QA & Performance
Ensure the app is stable, performant, and ready for review.

**Outcomes**
- Unit/UI tests for critical flows.
- Device and simulator matrix validated.
- Performance profiling issues resolved.

# [A8] App Store Assets & Metadata
Prepare marketing assets and metadata for App Store Connect.

**Outcomes**
- App icon and screenshots complete.
- Store listing copy drafted.
- Pricing and release strategy set.

# [A9] App Store Connect & Distribution
Configure the app record, signing, and submission workflow.

**Outcomes**
- App Store Connect record created.
- Provisioning and signing validated.
- TestFlight pipeline established.

# [A10] Apple Review & Release
Submit for review and respond to reviewer feedback.

**Outcomes**
- Submission ready build uploaded.
- Review responses prepared.
- Release monitoring plan in place.

# [A11] ChapGPT App Store Release (Parallel Track)
Treat this as a separate distribution track and mirror metadata/assets.

**Outcomes**
- ChapGPT submission requirements gathered.
- Parallel checklist created.
- Submission completed after Apple release (or in parallel as allowed).

# [A12] Operational Readiness & Post-Launch
Prepare support, analytics, and iteration workflows.

**Outcomes**
- Support channels and FAQs drafted.
- Analytics dashboards configured (if used).
- Post-launch iteration cadence scheduled.
